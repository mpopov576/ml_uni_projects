# ml_uni_projects

**Projects from the course “Machine Learning and Extracting Patterns from Data” at the Faculty of Mathematics and Informatics, Sofia University.**

<details>
<summary><strong>Standards for Model Reports</strong></summary>

1. **Each row represents a hypothesis (a trained model).**
   Columns are divided into:

   * Hyperparameters
   * Evaluation metrics (max 3)

2. **Baseline model (first row):**

   * Existing deployed model OR
   * Greedy statistical model (e.g. majority class classifier)

3. **Metrics columns:**

   * Show metric values
   * Include % change compared to baseline

4. **Comments column:**

   * Interpretation of results
   * May include error analysis

5. **Best model identification:**

   * Clearly state which model is best and why

6. **Required visualizations:**

   * Train vs validation metric
   * Train vs validation loss (if applicable)

7. **Table formatting:**

   * Not raw pandas output
   * Use colors, highlights, bold

8. **Order matters:**

   * Do NOT sort models after experiments
   * Keep chronological order

9. **Keep it readable:**

   * Avoid very wide tables
   * Best model should be obvious in 1–2 seconds

10. **Optional:**

* Separate sheet with 4–5 correct/incorrect predictions
* Helps manage client expectations

</details>
